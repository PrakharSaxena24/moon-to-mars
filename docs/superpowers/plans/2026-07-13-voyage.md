`node verify.js` green at every hand-off; existing single-day behavior byte-compatible when the manifest is
fully aboard (pinned); Live mode untouched (fishday-only, carry-canonical).

## Key pinned interfaces
- `carryState(plan)` → `{itemId: {seg: 'aboard'|'missing'|'late'}}`-shaped read-off; exported on `api`;
  `daySchedule` internally consults it — NO signature change; all-aboard ⇒ outputs byte-identical to today
  (verify pins this on the canonical plan).
- `plan.guests[i] = {id, name{en,jp}, vip, party}`; `overrides.buddies {guestId: pid}`; buddy task ids
  `t_v_star_<guestId>`, `t_v_esc_l_<guestId>`, `t_v_esc_d_<guestId>`.
- New i18n key families: `gd_*` (guest/care), `snd*` (sound toggle), day labels for load/voyage; all EN+JP.
- Sound API: `window.PRS_SOUND = {enabled, toggle(), cue(name), ambient(seg, dayK)}` — app calls are
  fire-and-forget and null-safe (the game must run identically without sound.js, like sprites).
- Matrix targets (spec §4): frame~12 · load~11 · voyage~12 · arrival~12 · ops~14 · fishday~30(heaviest) ·
  return~9 = 100; exact landed values become verify PINS at W4.
